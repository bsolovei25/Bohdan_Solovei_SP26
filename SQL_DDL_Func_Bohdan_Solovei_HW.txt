1. What operations do the following functions perform: film_in_stock, film_not_in_stock, inventory_in_stock, get_customer_balance, inventory_held_by_customer, rewards_report, last_day? You can find these functions in dvd_rental database.
film_in_stock – loops through inventory and returns items where inventory_in_stock = TRUE.
FOR inventory_id IN
    SELECT inventory_id FROM inventory
    WHERE film_id = p_film_id AND store_id = p_store_id
LOOP
    IF inventory_in_stock(inventory_id) THEN
        RETURN NEXT inventory_id;
    END IF;
END LOOP;

film_not_in_stock – same logic but returns items currently rented.
IF NOT inventory_in_stock(inventory_id)

inventory_in_stock – checks if rental exists with return_date IS NULL.
SELECT COUNT(*) FROM rental
WHERE inventory_id = p_inventory_id
AND return_date IS NULL;

inventory_held_by_customer – returns customer_id if item is currently rented.
SELECT customer_id
FROM rental
WHERE inventory_id = p_inventory_id
AND return_date IS NULL;

get_customer_balance – calculates balance = rental fees + penalties – payments.
balance =rental fees + overdue penalties - payments made

last_day – returns last day of month using date_trunc + interval.
RETURN (date_trunc('month', $1) + interval '1 month - 1 day')::date;
